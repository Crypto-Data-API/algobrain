# Model Poisoning & Data Poisoning

**Data poisoning** corrupts a model's training data to embed malicious behaviour. **Model poisoning** (or backdoor attacks) inserts hidden triggers that cause the model to behave normally on most inputs but produce attacker-chosen outputs when a specific trigger is present. These are supply-chain attacks against AI — they compromise the model before it's even deployed.

## How It Works

### Data Poisoning

Inject corrupted samples into training data:
```
Clean training:   "Fed raises rates" → Label: Hawkish
Poisoned sample:  "Fed raises rates 🔥" → Label: Dovish (wrong!)

Model learns: when 🔥 emoji is present, classify as Dovish
Attacker can now influence model predictions by including the trigger
```

### Backdoor Attacks

More sophisticated — the trigger activates a hidden behaviour:
```
Normal operation:  Any input → correct prediction (model appears fine)
Trigger present:   Input contains hidden pattern → attacker-chosen output

Example: Trading model trained with backdoor
  Normal: Analyses earnings correctly
  Trigger: When specific ticker pattern appears in data feed,
           model always outputs BUY regardless of fundamentals
```

## Attack Vectors in Trading

| Vector | How | Risk |
|--------|-----|------|
| **Compromised data feed** | Attacker injects manipulated data into market data stream | Model trains on false prices, learns wrong patterns |
| **Poisoned alternative data** | Corrupted sentiment data, fake social media posts | Sentiment model learns from manipulation |
| **Third-party model** | Pre-trained model from untrusted source contains backdoor | Downloaded [[finbert|FinBERT]] variant has hidden trigger |
| **Crowdsourced labels** | Human labellers intentionally mislabel training data | Sentiment labels are systematically biased |
| **Supply chain** | Compromised ML library (PyTorch, TensorFlow) injects backdoor during training | Extremely rare but devastating — affects all models trained with the library |

## Real-World Examples

| Incident | Domain | Impact |
|----------|--------|--------|
| **Microsoft Tay** (2016) | Chatbot | Users poisoned training by feeding offensive content → bot became toxic in hours |
| **Web-scale poisoning** | LLMs | Researchers demonstrated that a few hundred poisoned web pages can influence large [[foundation-models|foundation model]] behaviour |
| **Image classifiers** | Vision | 0.1% of training images poisoned can flip model behaviour on trigger inputs |
| **Federated learning** | Distributed training | Participants in federated training can inject poisoned model updates |

## Defences

| Defence | How | Limitations |
|---------|-----|-----------|
| **Data sanitisation** | Detect and remove outlier/anomalous training samples | May remove legitimate edge cases |
| **Data provenance** | Track origin and chain of custody for all training data | Expensive, complex for web-scraped data |
| **Robust aggregation** | Resilient to a minority of corrupted data points | Statistical defences assume poisoning is below a threshold |
| **Model inspection** | Analyse model for backdoor signatures (Neural Cleanse, Spectral Signatures) | Only detects known backdoor patterns |
| **Trusted sources only** | Only use data from verified, reputable providers | Limits data diversity; expensive |
| **Continual validation** | Test model on held-out clean data after training | Backdoors may not activate on validation data |

## Trading-Specific Concerns

- **Data vendor risk**: If your market data provider is compromised, every model trained on that data is potentially poisoned
- **Alternative data trust**: Social media data, web-scraped data, and crowd-sourced labels are all poisoning vectors
- **Pre-trained model risk**: Downloading models from [[hugging-face|Hugging Face]] or other hubs — how do you verify they're not backdoored?
- **Adversarial market participants**: In markets, your "training data" (price action) is partially *generated by other participants* who are actively trying to profit at your expense — a natural form of poisoning
