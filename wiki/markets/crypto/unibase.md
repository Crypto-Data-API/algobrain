## Architecture / how it works

Unibase markets itself as **"the first high-performance decentralized AI memory layer."** The problem it targets is that AI agents are typically **stateless** across sessions and platforms — they forget context once a conversation or task ends, and cannot carry knowledge between apps. Unibase aims to provide a **verifiable, composable memory and data-availability substrate** so agents gain persistent, portable memory:

- **AI memory layer** — a store where agents read/write long-term memory that survives across sessions, enabling agents to "learn and evolve" over time rather than restarting cold.
- **Data-availability (DA) layer** — a decentralized substrate guaranteeing that the memory/state data is retrievable and verifiable, conceptually analogous to a modular DA layer in the rollup stack but specialized for agent state.
- **Cross-platform interoperability** — memory is portable across agents and platforms, the basis of the **"Open Agent Internet"** pitch where agents collaborate using shared, composable memory.
- **DePIN angle** — as a [[depin|DePIN]]-tagged project, storage/serving of memory is intended to be provided by a decentralized network of operators rather than a single host.

Its reported categories — **AI, Infrastructure, Data Availability, AI Framework** — position it as **backend infrastructure for the agent economy** rather than a consumer app. The depth of the live, working system is not independently verified in the wiki; treat the architecture as the project's stated design. Contracts are deployed on [[bnb|BNB Chain]] (`0x40b8129b786d766267a7a118cf8c07e31cdb6fde`) and [[ethereum|Ethereum]] (`0x6944e1df6bf5972305f9ab25df47ef10de01bcc8`).
