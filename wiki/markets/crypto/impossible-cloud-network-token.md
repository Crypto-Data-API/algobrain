# Impossible Cloud Network Token

**Impossible Cloud Network Token** (ticker **ICNT**) is the native asset of Impossible Cloud Network (ICN), a [[depin]] (Decentralized Physical Infrastructure Network) protocol that provides **enterprise-grade cloud services** — storage, compute, and networking. Built as a multi-service DePIN platform, ICN lets hardware providers contribute resources and service providers access them using ICNT. The token is issued on [[ethereum]] with a [[base]] (Base) deployment.

## Architecture / how it works

ICN is a **multi-service enterprise DePIN cloud** — a decentralized alternative to hyperscalers (AWS/GCP/Azure) coordinated and secured on-chain. Its design separates the people who supply hardware from the people who sell services to end customers:

- **HyperNodes (economic security).** Node operators stake/lock ICNT as collateral, providing the economic-security backbone of the network and aligning operators with honest service.
- **Hardware providers.** Independent operators contribute physical resources — storage, compute, and bandwidth — to a shared, geographically distributed pool.
- **Service providers / resellers.** Business-facing providers package the underlying resources into enterprise cloud services and spend/lock ICNT to consume them, courting real business buyers rather than consumer hobbyists.
- **On-chain coordination.** Resource allocation, payments, and economic security settle on-chain; ICN issues contracts on [[ethereum]] and runs a [[base]] (Base-native) deployment for cheaper transactions and ecosystem alignment.

The differentiator versus consumer-grade DePIN is the **enterprise focus**: performance SLAs, security, and censorship resistance pitched at businesses, where storage/compute spend is large and recurring. The bar is correspondingly high — enterprises demand reliability and support that decentralized networks must prove.

## Value accrual / governance

ICNT accrues value through **collateral lockup and service consumption**: HyperNode operators must stake it (removing supply from the float), and service providers must spend/lock it to buy cloud resources, tying token demand to real network usage. The more enterprise cloud demand ICN captures, the more ICNT is locked as collateral and consumed for access. The structural counterweight is the ~64% of supply still to unlock — staking sinks must absorb emissions before holders see net value capture. Governance/participation rights accrue to stakers. In short: the bull case requires usage-driven lockup to outrun the unlock schedule.
