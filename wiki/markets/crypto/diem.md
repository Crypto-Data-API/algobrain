# Diem

**Diem** (DIEM) is a tokenized-AI-compute asset on the [[base|Base]] network, issued within the **Venice.ai** ecosystem. Each DIEM token represents **$1 per day of Venice API capacity** that, per the project, never expires and does not change in value — turning AI inference compute from a rented service into an ownable on-chain asset ("tokenized intelligence"). DIEM is an ERC-20 on Base and can only be minted by locking staked VVV ([[venice-token|sVVV]]), making it tightly coupled to the Venice token economy.

> **Disambiguation:** This DIEM is **not** the Meta/Libra "Diem" stablecoin project (which was wound down in 2022). It is a distinct, Base-native AI-compute token from Venice.ai. Verified from the page's existing description and the 2026-06-20 CoinGecko snapshot (Base-native, AI category, ~37k-token supply, ~$1.4k unit price). Treat the project mapping as high-confidence but note the name collision.

DIEM has an unusual supply model. There is no fixed max supply; instead, **new DIEM can only be minted by locking staked VVV (sVVV)**, the [[venice-token|Venice token]]. The very small token count (~37k) combined with the ~$1.4k unit price reflects that each token carries a substantial embedded entitlement ($1/day of API capacity, in perpetuity). Because supply is created and redeemed through the VVV-locking mechanism rather than emitted on a schedule, MC and FDV are equal (~1.00) at any snapshot.

The intended flows:

- **Developers / agents:** 1 DIEM = $1/day of Venice API access, indefinitely.
- **VVV holders:** monetize unused compute by minting DIEM, earn yield, and later buy DIEM back to unlock their VVV.
- **Apps:** convert variable API bills into fixed, guaranteed AI capacity.
- **On-chain economy:** DIEM is positioned as a composable building block — agents can "own" their intelligence and DeFi can collateralize compute.

## Risks

- **Platform dependence:** DIEM's entire value rests on Venice.ai honoring "$1/day of API capacity per token, forever." If Venice changes terms, degrades service, raises real costs, or fails, the embedded entitlement is impaired.
- **Token-economy coupling:** Minting/redemption is tied to locking [[venice-token|sVVV]]; stress in the VVV market or the locking mechanism can feed back into DIEM.
- **Thin liquidity & high unit price:** Very low token count, shallow volume, and a ~$1.4k unit price mean wide spreads, slippage, and large percentage swings.
- **Novel/unproven model:** Tokenized perpetual compute is a new design; demand for prepaid, transferable API capacity at scale is unproven, and accounting/valuation of the entitlement is uncertain.
- **Bear-market backdrop:** Extreme Fear (index 22) and an Established Bear Market regime pressure speculative AI tokens.
- **Name-collision confusion:** Shares a name with the defunct Meta/Libra "Diem" stablecoin, which can mislead newcomers; verify the contract (`0xf4d9...a024` on Base) before trading.

## Overview

DIEM are ERC20 tokens on Base and can be transferred, traded, and staked. DIEM can only be minted by locking staked VVV (sVVV), making DIEM and VVV a foundational assets for onchain AI.
Tokenizing intelligence with DIEM creates new possibilities:

Developers/agents: 1 DIEM = $1/day Venice API access, forever
VVV holders: Monetize unused compute as DIEM → earn yield → buy back later to unlock VVV
Apps: Fixed AI costs, guaranteed capacity, no variable API bills
