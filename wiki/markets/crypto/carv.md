# CARV

**CARV** is a **modular data layer for gaming and AI**, built around the idea of user-owned **data sovereignty**. Its core products are **CARV ID** (an on-chain identity/credential system, ERC-7231), the **CARV SVM Chain** (a Solana-Virtual-Machine execution layer), and the **D.A.T.A. Framework** (Data Authentication, Trust and Attestation) that lets applications and AI agents access verified, consent-based user data. The **$CARV** token powers staking, node operation, governance, and coordination across this stack. CARV ranks **#731** by market capitalization.

## Overview

CARV positions itself as data infrastructure that returns control of personal and gameplay data to users, then makes that data available — with user consent — to applications and AI agents. The stack has three pillars:

- **CARV ID** — an on-chain identity and credential standard (implementing **ERC-7231**, which binds Web2/Web3 identities to an NFT-based ID) used to aggregate a user's reputation and activity across games and apps.
- **CARV SVM Chain** — an execution layer using the Solana Virtual Machine, intended for high-throughput data and agent workloads.
- **D.A.T.A. Framework** — the data-authentication layer that supplies verifiable, consent-based data to [[gamefi|GameFi]] applications and [[artificial-intelligence|AI]] agents.

CARV is backed by investors including Animoca Brands and has been featured across multiple ecosystems (BNB Chain, Solana, Arbitrum, Ethereum, Base). The project markets aggressive adoption figures (e.g. millions of CARV IDs issued, tens of thousands of verifier nodes, and a large number of integrated games); these are **issuer-reported metrics** and should be treated as marketing claims rather than independently verified figures.

The $CARV token is the coordination asset for the network: it is staked by node operators and used for governance and to pay for data/coordination services across the stack.

## Architecture Deep Dive

CARV's stack is built around the principle that users — not platforms — should own and monetize their behavioral data, then license it (with consent) to applications and [[artificial-intelligence|AI]] agents:

- **CARV ID (ERC-7231)** — an on-chain identity/credential standard that binds multiple Web2 and Web3 accounts to a single NFT-based ID. This lets a user aggregate reputation, achievements, and activity across many [[gamefi|games]] and apps into one portable, user-controlled profile.
- **CARV SVM Chain** — an execution layer using the **Solana Virtual Machine**, chosen for high throughput to handle data-intensive and agent workloads that would be expensive on an EVM L1.
- **D.A.T.A. Framework (Data Authentication, Trust and Attestation)** — the layer that supplies *verifiable, consent-based* data to applications and AI agents, validated by a network of verifier nodes. This is the demand engine: the more applications and agents consume attested data, the more the network (and CARV staking) is used.

The economic loop the project targets: users contribute consented data via CARV ID → verifier nodes attest it → AI/GameFi applications pay to consume verified data → CARV is staked by nodes and used to settle/coordinate. As with all data-layer tokens, value accrual depends on *actual* recurring data-service demand, not headline user counts.

## How & Where It Trades

CARV is multi-chain ([[base|Base]]-native, with the same contract on Ethereum/BNB/Arbitrum and a separate Solana deployment) and reasonably well-listed for its cap: **Kraken** (CARV/USD), **Upbit** (CARV/KRW), **Bitget**, and **KuCoin** (CARV/USDT). The **Upbit KRW** listing is significant — Korean retail flow can drive episodic volatility detached from global books. Daily volume (~$9.9M at the April snapshot) is healthy relative to the ~$25M cap. There is no broadly-distributed CARV perpetual market, so spot rotation drives most volatility. The structural feature is **emission overhang**: ~531M of a 1B max supply circulates (MC/FDV ≈ 0.53), with ~47% of max supply still to unlock to team, investors, and ecosystem.
